Level 1: React Basics

1. Introduction to React

Definition:

React is an open-source JavaScript library used for building dynamic, fast, and interactive user interfaces, primarily for single-page applications (SPAs). It focuses on creating reusable UI components.

Why is it Important:

React allows developers to build large, complex applications where the UI updates efficiently as data changes. It's lightweight, component-based, and backed by a strong ecosystem (like Redux, React Router, etc.), making it ideal for scalable projects.

How it Works (Concept Explanation):

React works by maintaining a lightweight representation of the DOM in memory (called the Virtual DOM). When a change occurs, React updates only the parts of the real DOM that actually changed — leading to faster UI updates.

Code Example:

import React from 'react';
  
function App() {
  return <h1>Hello, World!</h1>; 
} 
  
export default App;

2. Setting Up Environment

Definition:

Setting up the environment means installing all the necessary tools (Node.js, npm, React, a code editor) to start developing a React application.

Why is it Important:

A correct setup ensures the project runs smoothly, supports modern JavaScript (ES6+), and allows integration with development tools (like hot reloading, linters, etc.)

How it Works (Concept Explanation):

We install Node.js (includes npm - Node Package Manager) and then use Create React App (CRA), a command-line tool to bootstrap a ready-to-use React project.

Node.js: https://nodejs.org/en/download

Code Example:

npx create-react-app my-app
cd my-app
npm start

3. JSX (JavaScript XML)

Definition:

JSX is a syntax extension for JavaScript that allows us to write HTML-like code inside JavaScript files. React components are typically written using JSX.

Why is it Important:

JSX makes code easier to read and write by visually resembling the final HTML structure while still having the power of JavaScript embedded.

How it Works (Concept Explanation):

Browsers don't understand JSX directly — during the build process, JSX gets transpiled into regular JavaScript using tools like Babel.

Code Example:

// This is JSX code:
const element = <h1>Hello, JSX!</h1>;

// This transpiles to JavaScript:
const element = React.createElement('h1', null, 'Hello, JSX!');

4. Components (Function and Class)

Definition:

Components are the building blocks of a React app. They can be either Functional Components or Class Components and are reusable, independent pieces of UI.

Why is it Important:

Breaking a UI into smaller, manageable components improves reusability, readability, and maintainability.

How it Works (Concept Explanation):

  • Functional Components: Simple JavaScript functions that return JSX.
  • Class Components: ES6 classes that extend React.Component and can hold their own state.

Code Example:

// Functional Component 
function Welcome() { 
  return <h1>Welcome!</h1>; 
} 

// Class Component 
class WelcomeClass extends React.Component { 
  render() { 
    return <h1>Welcome from Class!</h1>; 
  } 
}

5. Props (Properties)

Definition:

Props are inputs to a React component. They are passed as attributes from a parent component to a child and are read-only.

Why is it Important:

Props allow components to be dynamic and customizable without changing their internal logic.

How it Works (Concept Explanation):

Think of props like function parameters. When using a component, you pass values (props) into it.

Code Example:

function Greet(props) { 
  return <h1>Hello, {props.name}!</h1>; 
} 
  
// Usage 
<Greet name="John" />

6. State Management with useState

Definition:

useState is a React Hook that allows functional components to create, store, and update internal state variables. It gives components the ability to dynamically change their UI based on user interactions or other events.

Why is it Important:

Without useState, functional components would be static — no interaction, no dynamic changes. With useState, your app can:

  • Track user inputs.
  • Update the UI when something changes (like button clicks, form entries, etc.).
  • Manage dynamic values easily without needing a class component.

It makes functional components just as powerful (or even more!) than class components.

How it Works (Concept Explanation):

Think of useState like a special variable that remembers values between function calls.

  • When you call useState(initialValue), it gives you two things:
    • A state variable (current value).
    • A setter function (to update the value).

When you call the setter, React:

  • Updates the value.
  • Renders the component again with the new value.

📦 Analogy: Imagine a box (state) where you keep a counter value. Whenever you update the counter (setter), the box automatically updates the display (UI) without you manually reloading anything.

Code Example:

import { useState } from 'react'; 
  
function Counter() { 
  const [count, setCount] = useState(0); // Initial value = 0 
  
  function increment() { 
    setCount(count + 1); // Update state 
  } 
  return ( 
    <div> 
      <p>Clicked {count} times</p> 
      <button onClick={increment}>Click Me</button> 
    </div> 
  ); 
} 
export default Counter;

State in React:

Definition: State is a built-in object that stores property values that belong to the component. State changes trigger UI updates.

Why is it Important: State is essential for dynamic, interactive applications where the UI needs to change based on user input, server responses, etc.

How it Works: State is managed inside the component using the useState Hook (in functional components) or inside this.state (in class components).

Code Example:

import { useState } from 'react'; 
  
function Counter() { 
  const [count, setCount] = useState(0); 
  
  return ( 
    <div> 
      <p>Clicked {count} times</p> 
      <button onClick={() => setCount(count + 1)}>Click me</button> 
    </div> 
  ); 
}

Difference between "State in React" and "State Management with useState"

Term Meaning Example
State General idea of storing dynamic, changeable data inside a component. "User's login status", "Input field text"
State Management with useState A React Hook used to create and update state inside functional components. const [count, setCount] = useState(0);

7. Handling Events

Definition:

Event handling in React is similar to handling events in DOM elements but with some syntactic differences. React events are named using camelCase.

Why is it Important:

Interactivity in a React app is impossible without handling user events like clicks, typing, submitting forms, etc.

How it Works (Concept Explanation):

We attach event handlers to JSX elements, and React manages event delegation under the hood.

Code Example:

function Button() { 
  function handleClick() { 
    alert('Button Clicked!'); 
  } 
  
  return <button onClick={handleClick}>Click Me</button>; 
}

8. Conditional Rendering Techniques

Definition:

Conditional rendering in React means showing different UI elements or components based on certain conditions (like if-else logic for the UI).

Why is it Important:

In real-world apps, UIs often change depending on user actions, authentication status, API responses, etc. Conditional rendering allows you to create dynamic and interactive user experiences.

How it Works (Concept Explanation):

You can use JavaScript conditional logic directly inside JSX to decide what to display.

Common techniques:

  • if / else
  • Ternary operator (condition ? true : false)
  • Logical && (short-circuiting)

Code Examples:

// Using if-else outside JSX:
function Greeting({ isLoggedIn }) { 
  if (isLoggedIn) { 
    return <h1>Welcome back!</h1>; 
  } 
  return <h1>Please sign up.</h1>; 
}
// Using Ternary inside JSX:
function Greeting({ isLoggedIn }) { 
  return ( 
    <h1>{isLoggedIn ? 'Welcome back!' : 'Please sign up.'}</h1> 
  ); 
}
// Using Logical && (Show only if condition true):
function Notification({ unreadMessages }) { 
  return ( 
    <div> 
      {unreadMessages.length > 0 &&  
        <h2>You have {unreadMessages.length} unread messages.</h2> 
      } 
    </div> 
  ); 
}

9. List Rendering using .map()

Definition:

List rendering in React means creating a list of elements/components dynamically by iterating over an array.

Why is it Important:

Most real apps involve dynamic data — like lists of products, users, posts, etc. .map() helps display multiple items from a dynamic data source cleanly.

How it Works (Concept Explanation):

  • Take an array.
  • Use .map() to loop through each item.
  • Return a new React element for each item.

Code Example:

function NameList() { 
  const names = ['Alice', 'Bob', 'Charlie']; 
  
  return ( 
    <ul> 
      {names.map(name => <li key={name}>{name}</li>)} 
    </ul> 
  ); 
}

10. Keys in Lists

Definition:

Keys are special props you need to assign when rendering lists in React. They help React identify which items have changed, are added, or removed.

Why is it Important:

Without keys, React can't efficiently re-render lists. Proper keys lead to better performance and bug-free rendering.

How it Works (Concept Explanation):

Keys should be:

  • Unique among siblings.
  • Stable (doesn't change between renders).

Code Example:

const items = ['Apple', 'Banana', 'Cherry']; 
  
return ( 
  <ul> 
    {items.map(item => <li key={item}>{item}</li>)} 
  </ul> 
);

11. Basic Form Handling

Definition:

Form handling in React means capturing user input and updating the component's state based on that input.

Why is it Important:

Forms are the backbone of user interaction — login forms, registration, settings, etc. Controlled components (React forms) make it easy to track input values and validate them.

How it Works (Concept Explanation):

In a controlled component:

  • The input's value is linked to the component's state.
  • Changes to the input are handled by updating the state with onChange.

Code Example:

import { useState } from 'react'; 
  
function SimpleForm() { 
  const [name, setName] = useState(''); 
  
  function handleChange(event) { 
    setName(event.target.value); 
  } 
  
  function handleSubmit(event) { 
    event.preventDefault(); 
    alert(`Submitted Name: ${name}`); 
  } 
  
  return ( 
    <form onSubmit={handleSubmit}> 
      <input type="text" value={name} onChange={handleChange} /> 
      <button type="submit">Submit</button> 
    </form> 
  ); 
}

12. Inline and External Styling

Definition:

React allows you to style components inline (inside JSX) or by using external CSS files.

Why is it Important:

Good styling improves UX and separates presentation from logic. Understanding both methods is crucial for professional UI development.

How it Works (Concept Explanation):

  • Inline styling: Pass a style object directly inside JSX.
  • External styling: Import a CSS file and use class names.

Code Examples:

// Inline styling:
const headingStyle = { 
  color: 'blue', 
  fontSize: '24px' 
}; 
  
function InlineExample() { 
  return <h1 style={headingStyle}>Hello World</h1>; 
}
// External styling:
/* styles.css */  
.title {  
  color: red;  
  font-size: 24px;  
} 

// Example:
import './styles.css';   // ✅ External CSS file imported 
  
function ExternalExample() { 
  return ( 
    <h1 className="title">Hello World</h1>  // ✅ Applying a className from External CSS 
  ); 
}
// Both Inline + External:
import './styles.css';   
  
function MixedExample() { 
  return ( 
    <h1  
      className="title"  
      style={{ color: 'blue', fontSize: '32px' }}  // ✅ Inline Style 
    > 
      Hello World 
    </h1> 
  ); 
}
