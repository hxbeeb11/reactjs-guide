1. Introduction to React Hooks

Definition:

Hooks are special functions in React that let you "hook into" React features (like state, lifecycle methods, context) without writing a class.

Why It's Important:

  • Simplifies components — no more messy classes.
  • Code reusability — custom hooks share logic easily.
  • Modern standard — Hooks are the future of React, replacing many older patterns.

How it Works:

Hooks like useState, useEffect, useContext, etc., manage component logic directly inside functional components.

Code Example:

import { useState } from 'react'; 
  
function Counter() { 
  const [count, setCount] = useState(0); // Hook: useState 
  
  return ( 
    <div> 
      <p>Count: {count}</p> 
      <button onClick={() => setCount(count + 1)}>Increment</button> 
    </div> 
  ); 
}

2. useEffect Hook for Side Effects

Definition:

The useEffect hook lets you perform side effects (data fetching, subscriptions, manually changing the DOM) after rendering.

Why It's Important:

  • Handles lifecycle events (like componentDidMount, componentDidUpdate, componentWillUnmount) inside functional components.

How it Works:

useEffect takes two arguments: a function to run, and a dependency array.

  • Empty array [] → run once (after mount).
  • No array → run after every render.
  • Specific variables [data] → run when data changes.

Code Example:

import { useEffect, useState } from 'react'; 
  
function FetchUser() { 
  const [user, setUser] = useState(null); 
  
  useEffect(() => { 
    fetch('https://api.example.com/user') 
      .then(res => res.json()) 
      .then(data => setUser(data)); 
  }, []); // Empty dependency array → runs once 
  
  return <div>{user ? user.name : 'Loading...'}</div>; 
}

3. Controlled vs Uncontrolled Components

Definition:

Controlled Components:

React controls the form elements through state.

Uncontrolled Components:

DOM itself manages the form element state.

Why It's Important:

Understanding both helps in building professional, predictable forms. Controlled is more common for large apps, while uncontrolled is simpler for small cases.

How it Works:

  • Controlled: value and onChange controlled by React.
  • Uncontrolled: Use ref to directly access DOM value.

Code Example:

Controlled Component:
function ControlledInput() { 
  const [value, setValue] = useState(''); 
  return ( 
    <input value={value} onChange={(e) => setValue(e.target.value)} /> 
  ); 
}
Uncontrolled Component:
import { useRef } from 'react'; 
  
function UncontrolledInput() { 
  const inputRef = useRef(); 
   
  function handleSubmit() { 
    console.log(inputRef.current.value); 
  } 
  
  return ( 
    <> 
      <input ref={inputRef} /> 
      <button onClick={handleSubmit}>Submit</button> 
    </> 
  ); 
}

4. Form Handling Deep Dive

Definition:

Form handling means capturing user input, validating, and processing form submissions properly inside a React app.

Why It's Important:

Professional apps must handle validations, dynamic inputs, error handling, and submission efficiently.

How it Works:

Capture input with useState, validate, and submit. For complex forms, libraries like Formik or React Hook Form are used.

Code Example:

function SimpleForm() { 
  const [email, setEmail] = useState(''); 
  
  function handleSubmit(e) { 
    e.preventDefault(); 
    console.log(email); 
  } 
  
  return ( 
    <form onSubmit={handleSubmit}> 
      <input 
        type="email" 
        value={email} 
        onChange={(e) => setEmail(e.target.value)} 
        required 
      /> 
      <button type="submit">Submit</button> 
    </form> 
  ); 
}

5. Lifting State Up and Data Flow

Definition:

Lifting state up means moving state to a common parent component to share it between multiple child components.

Why It's Important:

  • Ensures single source of truth.
  • Solves data synchronization issues between sibling components.

How it Works:

Move the useState hook to the closest common ancestor and pass data + set functions down as props.

Code Example:

Before Lifting State (problem):
function Child() { 
  const [input, setInput] = useState(''); 
  return <input value={input} onChange={(e) => setInput(e.target.value)} />; 
}
After Lifting State (solution):
function Parent() { 
  const [input, setInput] = useState(''); 
  
  return ( 
    <> 
      <Child input={input} setInput={setInput} /> 
      <Child input={input} setInput={setInput} /> 
    </> 
  ); 
} 
  
function Child({ input, setInput }) { 
  return <input value={input} onChange={(e) => setInput(e.target.value)} />; 
}

6. React Router (react-router-dom)

Definition:

React Router is the standard routing library for React applications. It lets you navigate between different components/pages without refreshing the browser.

Why It's Important:

  • Builds multi-page experiences inside single-page applications (SPAs).
  • Provides dynamic routing, URL parameters, and navigation control.
  • Essential for any real-world app (e.g., dashboards, websites, admin panels).

How it Works:

You define routes (URL paths) linked to specific components. React Router watches the URL and renders the matching component without reloading.

Code Example:

npm install react-router-dom 
// App.js 
import { BrowserRouter, Routes, Route } from 'react-router-dom'; 
import Home from './Home'; 
import About from './About'; 
  
function App() { 
  return ( 
    <BrowserRouter> 
      <Routes> 
        <Route path="/" element={<Home />} /> 
        <Route path="/about" element={<About />} /> 
      </Routes> 
    </BrowserRouter> 
  ); 
}

7. Routes and Link

Definition:

Routes:

Define which components to render based on URL paths.

Link:

Navigate between routes without full page reload.

Why It's Important:

  • Enables SPA navigation while keeping browser history working normally.
  • SEO-friendly for client-side routing.

How it Works:

  • Use <Routes> to define all available paths.
  • Use <Link> instead of <a> for internal navigation.

Code Example:

import { Link, Routes, Route } from 'react-router-dom'; 
import Home from './Home'; 
import About from './About'; 
  
function Navbar() { 
  return ( 
    <> 
      <Link to="/">Home</Link> | <Link to="/about">About</Link> 
    </> 
  ); 
} 
  
function App() { 
  return ( 
    <> 
      <Navbar /> 
      <Routes> 
        <Route path="/" element={<Home />} /> 
        <Route path="/about" element={<About />} /> 
      </Routes> 
    </> 
  ); 
}

8. useNavigate for Redirection

Definition:

useNavigate() is a React Router hook to programmatically change the route inside your JavaScript code.

Why It's Important:

  • Automatically redirect users after actions like login, logout, form submit, etc.
  • No manual clicking needed; navigation happens inside event handlers.

How it Works:

Call navigate(path) inside an event (like form submission or button click).

Code Example:

import { useNavigate } from 'react-router-dom'; 
  
function Login() { 
  const navigate = useNavigate(); 
  
  function handleLogin() { 
    // Assume login success 
    navigate('/dashboard'); 
  } 
  
  return ( 
    <button onClick={handleLogin}>Login</button> 
  ); 
}

9. Dynamic Routing with URL Params

Definition:

Dynamic routing lets you create routes with variable parts (called URL parameters), like /user/:userId.

Why It's Important:

  • Supports dynamic pages based on user/product/blog IDs.
  • Essential for dynamic web apps (e.g., Profile pages, Product pages).

How it Works:

Define dynamic segments with :paramName and read them with useParams().

Code Example:

// App.js 
import { BrowserRouter, Routes, Route } from 'react-router-dom'; 
import UserProfile from './UserProfile'; 
  
function App() { 
  return ( 
    <BrowserRouter> 
      <Routes> 
        <Route path="/user/:userId" element={<UserProfile />} /> 
      </Routes> 
    </BrowserRouter> 
  ); 
} 
  
// UserProfile.js 
import { useParams } from 'react-router-dom'; 
  
function UserProfile() { 
  const { userId } = useParams(); // userId from URL 
  return <h1>User ID: {userId}</h1>; 
}

10. Component Reusability and Composition

Definition:

Component Reusability:

Creating components that can be used multiple times with different data or behaviors.

Composition:

Combining smaller components together to form larger, complex UIs.

Why It's Important:

  • Saves time and effort (Write once, reuse everywhere).
  • Makes code cleaner, easier to test, and maintainable.
  • Real-world projects (like dashboards, forms, card grids) heavily depend on reusable and composed components.

How it Works:

  • Create generic components (e.g., Button, Card, Modal).
  • Pass props to customize behavior or appearance.
  • Use children prop or composition patterns to build complex UIs.

Code Example:

Reusable Button Component:
// Button.js 
function Button({ label, onClick, type = "button" }) { 
  return <button type={type} onClick={onClick}>{label}</button>; 
} 
  
export default Button;
Using the Button in Different Places:
import Button from './Button'; 
  
function App() { 
  return ( 
    <> 
      <Button label="Save" onClick={() => alert('Saved!')} /> 
      <Button label="Delete" onClick={() => alert('Deleted!')} type="submit" /> 
    </> 
  ); 
}
Composition Example with children:
function Card({ children }) { 
  return <div className="card">{children}</div>; 
} 
  
<Card> 
  <h2>Title</h2> 
  <p>Description here...</p> 
</Card>

11. Introduction to Context API

Definition:

The Context API is React's built-in way to share global data (like user info, theme, or language) between components, without passing props manually at every level.

Why It's Important:

  • Avoids "prop drilling" (passing props through many layers unnecessarily).
  • Simplifies global state management for small to medium apps.
  • Essential for handling authentication, themes, user preferences, etc.

How it Works:

  1. Create a Context.
  2. Provide a value at the top-level.
  3. Consume the value in nested components using useContext().

Code Example:

Creating and Providing Context:
import { createContext, useState } from 'react'; 
  
// 1. Create Context 
const UserContext = createContext(); 
  
// 2. Provider Component 
function UserProvider({ children }) { 
  const [user, setUser] = useState("John Doe"); 
  
  return ( 
    <UserContext.Provider value={{ user, setUser }}> 
      {children} 
    </UserContext.Provider> 
  ); 
} 
  
export { UserContext, UserProvider };
Consuming Context:
import { useContext } from 'react'; 
import { UserContext } from './UserProvider'; 
  
function Profile() { 
  const { user } = useContext(UserContext); 
  
  return <h1>Welcome, {user}</h1>; 
}
Usage in App:
import { UserProvider } from './UserProvider'; 
  
function App() { 
  return ( 
    <UserProvider> 
      <Profile /> 
    </UserProvider> 
  ); 
}
